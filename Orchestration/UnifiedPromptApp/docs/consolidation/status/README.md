# Status Notes

Weekly heartbeat updates for the Unified Prompt Orchestrator program. Create one Markdown file per reporting date using `YYYY-MM-DD.md` naming (e.g., `2024-11-15.md`) and commit it under this directory.

## Usage
1. Copy `status_template.md` to the desired date.
2. Fill in the sections during/after Friday status review.
3. Reference the execution plan (`../03_execution_plan.md`) if scope or timelines shift.

## Recommended Sections
| Section | Purpose |
| --- | --- |
| Phase Focus | Which phase(s) are active this week. |
| Highlights | Completed deliverables or milestones. |
| Blockers / Risks | Anything jeopardizing plan dates plus requested help. |
| Next Week | Committed actions with owners. |
| Metrics | Optional snapshot (tests passed, prompts reviewed, etc.). |

## Example Workflow
```bash
cp status_template.md 2024-11-15.md
# edit 2024-11-15.md with latest updates
git add 2024-11-15.md
```
